Temuan IDOR Pada Layanan Indosat

Zildan Security • May 19, 2026

BUG-BOUNTY

Writeup ini mendeskripsikan skenario IDOR (Insecure Direct Object Reference) pada alur request yang menggunakan identifier sebagai acuan data transaksi.

Ringkasan Temuan

Endpoint menerima perubahan identifier tanpa melakukan verifikasi kepemilikan object secara ketat. Akibatnya, user berpotensi melihat resource milik user lain.

Step To Reproduce

  1. Autentikasi sebagai user biasa.
  2. Capture request saat membuka detail data tertentu.
  3. Modifikasi nilai ID ke milik user lain.
  4. Observasi response yang menampilkan data di luar hak akses.

Dampak

Kebocoran data antar akun, berpotensi meningkatkan risiko penyalahgunaan informasi maupun eskalasi serangan lanjutan.

Rekomendasi

  • Gunakan server-side object ownership validation sebelum query data.
  • Hindari identifier yang dapat ditebak, gunakan random UUID.
  • Tambahkan negative test untuk skenario lintas akun pada QA security test.
